## v34 Target

The v34 goal is a default warm-start runtime that pushes the SBAN architecture further without pretending it is a transformer or live-current web model:

- preserve the CPU numeric guardrail profile and keep `numeric_backend=cpu` plus `score_threads=1` as the packaged release default
- import the v33 `v33_arch`, exact-prompt retrieval, deterministic general-knowledge, instruction-memory, and auto-improvement stream lessons
- add `v34_arch` as a larger warm-start profile for chat/retrieval experiments
- generate `data/sban_runtime_prewarm_v34.txt` and load it by default for chat and accelerator corpus use
- make the default chat commands work without passing `seed_path`, `open_seed_path`, or `knowledge_path`
- keep compatibility v34 seed, open-seed, and synthetic-knowledge files generated by the prewarm builder
- broaden stable coverage for general knowledge, real-world task triage, reasoning, Python, SQL, and Zig coding patterns
- test vocabulary sizes through `65536` buckets and keep the dense byte-vocab core unchanged until sparse indexing or table redesign is implemented
- keep current-fact and unsupported source-location prompts on explicit external-lookup or source-boundary answers
- update repo and installed SBAN skill every generation before commit and push
- trigger the v34 GitHub release suite and v34 tag release bundle workflow before calling the release complete

## v34 Commands

Generate the warm-start pack and vocab probe:

```bash
python scripts/build_v34_runtime_prewarm.py
python scripts/vocab_size_probe_v34.py
```

Run the local product gate:

```bash
zig build test
zig build -Doptimize=ReleaseFast
python scripts/ci_smoke_v34.py
```

Run the hosted-compatible release suite:

```bash
python scripts/run_v34_release.py --skip-cuda --benchmarks prefix,drift,probe,long_250k,long_1m
```

Run hosted long hardening with the explicit 20M guardrail on GitHub-hosted runners:

```bash
python scripts/run_v34_release.py --skip-cuda --benchmarks long_20m --skip-dialogue --skip-accel --skip-numeric-backend-probes --skip-status --hosted-long-20m-guardrail
```

Generate deliverables:

```bash
python scripts/make_v34_deliverables.py
python scripts/package_v34_demo.py --binary zig-out/bin/zig_sban.exe --platform windows_x86_64
```

## Interpretation Guardrails

- Treat v34 as a stronger static/offline warm-start runtime, not a live web oracle.
- Treat broader knowledge as generated pack coverage plus deterministic symbolic helpers; do not claim open-ended LLM-scale knowledge.
- Keep retrieval thresholds and explicit uncertainty/source boundaries strong enough to avoid plausible but unsupported answers.
- Keep the numeric benchmark story separate from the product warm-start story.
- Do not switch the dense core vocab size based only on lower collision rates; the dense order-table memory cost remains the blocker.
- For releases, verify local gates, commit and push, monitor GitHub CI, trigger the versioned release suite, push the tag, monitor release packaging, and verify release assets.
