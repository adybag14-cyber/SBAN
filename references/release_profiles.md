## v35 Target

The v35 goal is a self-maintainable auto-improving runtime surface: improve replies by rebuilding online/dataset-generated assets rather than adding more prompt-specific code branches.

- preserve the CPU numeric guardrail profile and keep `numeric_backend=cpu` plus `score_threads=1` as the packaged release default
- add `v35_arch` as the larger auto-learn profile for chat/retrieval experiments
- generate `data/sban_runtime_prewarm_v35.txt` and load it by default for chat and accelerator corpus use
- generate `data/sban_learned_reasoning_v35.txt` from dataset adapters and load it by default in free-mode chat
- record source counts and online fetch errors in `docs/results/v35/autolearn_manifest_v35.json`
- use `data/sban_cold_seed_v35.txt` for `prewarm_path=none`, with `knowledge_path`, `open_seed_path`, and `learned_path` disabled
- keep compatibility v35 seed, open-seed, and synthetic-knowledge files generated by the prewarm builder
- fix exact JSON slot preservation for name/age prompts
- add structured forget/delete session semantics and filler cleanup for values like `my dog is max now`
- validate learned syllogism and arithmetic prompts directly through `data/sban_learned_session_eval_v35.txt`
- keep current-fact and unsupported source-location prompts on explicit external-lookup or source-boundary answers
- test vocabulary sizes through `65536` buckets and keep the dense byte-vocab core unchanged until sparse indexing or table redesign is implemented
- update repo and installed SBAN skill every generation before commit and push
- trigger the v35 GitHub release suite and v35 tag release bundle workflow before calling the release complete

## v35 Commands

Generate the warm-start pack, learned corpus, and vocab probe:

```bash
python scripts/build_v35_runtime_prewarm.py --force-refresh
python scripts/vocab_size_probe_v35.py
```

Run the local product gate:

```bash
zig build test
zig build -Doptimize=ReleaseFast
python scripts/ci_smoke_v35.py
```

Run the hosted-compatible release suite:

```bash
python scripts/run_v35_release.py --skip-cuda --benchmarks prefix,drift,probe,long_250k,long_1m
```

Run hosted long hardening with the explicit 20M guardrail on GitHub-hosted runners:

```bash
python scripts/run_v35_release.py --skip-cuda --benchmarks long_20m --skip-dialogue --skip-accel --skip-numeric-backend-probes --skip-status --hosted-long-20m-guardrail
```

Generate deliverables:

```bash
python scripts/make_v35_deliverables.py
python scripts/package_v35_demo.py --binary zig-out/bin/zig_sban.exe --platform windows_x86_64
```

## Interpretation Guardrails

- Treat v35 as a stronger offline/runtime-updatable assistant, not a live web oracle.
- Treat broader knowledge as generated pack coverage plus learned retrieval examples plus deterministic symbolic helpers; do not claim open-ended LLM-scale knowledge.
- Treat the learned corpus as data-side training/prewarm: future improvement should regenerate `data/sban_learned_reasoning_v35.txt` from dataset adapters, not add more static prompt branches.
- Keep retrieval thresholds and explicit uncertainty/source boundaries strong enough to avoid plausible but unsupported answers.
- Keep the numeric benchmark story separate from the product auto-learn story.
- Do not switch the dense core vocab size based only on lower collision rates; the dense order-table memory cost remains the blocker.
- For releases, verify local gates, commit and push, monitor GitHub CI, trigger the versioned release suite, push the tag, monitor release packaging, and verify release assets.
